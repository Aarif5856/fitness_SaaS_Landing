üì¶ Installation Guide

Method 1: Local Development

To view the template locally on your computer:

  1. Extract the ZIP file to a folder on your computer
  2. Open the index.html file in any modern web browser
  3. That's it! No build tools or server required
Tip: For the best experience, use a local server. You can use Python's built-in server: python -m http.server 8000 then visit http://localhost:8000

Method 2: Web Hosting

To deploy to a web server:

  1. Upload all files to your web hosting via FTP/SFTP or cPanel File Manager
  2. Ensure index.html is in the root directory (or public_html folder)
  3. Maintain the folder structure (especially the public/ folder)
  4. Visit your domain to see the live site

Method 3: Static Hosting Services

This template works perfectly with:

  • Vercel: Connect your GitHub repo or drag & drop the folder
  • Netlify: Drag & drop deployment or Git integration
  • GitHub Pages: Push to a GitHub repo and enable Pages
  • Cloudflare Pages: Connect repository for automatic deployments
Note: The template includes a vercel.json configuration file for easy Vercel deployment.

üé® Customization Guide

1. Changing Colors

All colors are controlled via CSS variables in styles.css. Open the file and find the :root section at the top.

Primary Colors

:root {
    /* Primary gradient colors */
    --gradient-primary: linear-gradient(135deg, #6366f1 0%, #8b5cf6 100%);
    --primary-500: #6366f1;
    --primary-400: #818cf8;
    --accent-500: #8b5cf6;
}

To change the brand color, update these values. For example, for a blue theme:

--gradient-primary: linear-gradient(135deg, #3b82f6 0%, #2563eb 100%);
--primary-500: #3b82f6;
--accent-500: #2563eb;

Background Colors

--bg-primary: #09090f;      /* Main background */
--bg-secondary: #0d0d15;   /* Secondary sections */
--bg-card: rgba(255, 255, 255, 0.03);  /* Card backgrounds */

Text Colors

--text-primary: #ffffff;                    /* Main text */
--text-secondary: rgba(255, 255, 255, 0.8);  /* Secondary text */
--text-tertiary: rgba(255, 255, 255, 0.6);   /* Tertiary text */

2. Changing Fonts

The template uses system fonts by default. To add custom fonts (e.g., Google Fonts):

  1. Add the font link in index.html <head> section:
<link href="https://fonts.googleapis.com/css2?family=Poppins:wght@400;600;700&display=swap" rel="stylesheet">
  1. Update CSS variables in styles.css:
--font-display: 'Poppins', sans-serif;
--font-body: 'Poppins', sans-serif;

3. Updating Content

Text Content

Open index.html and search for the section you want to edit. All text is clearly marked with HTML comments like:

<!-- Hero Section -->
<!-- Features Section -->
<!-- Pricing Section -->

Logo Replacement

  1. Replace public/logos/fitpulse_logo.png with your logo
  2. Keep the same filename OR update the path in index.html: 
    <img src="public/logos/your-logo.png" alt="Your Brand" class="nav-logo-img">
  3. For best results, use a transparent PNG with height around 40-50px

Images

  • Hero Mockup: Replace hero-mockup.png with your app screenshot or device mockup (recommended: 1200x800px)
  • Feature Icons: Replace workout-icon.png, nutrition-icon.png, progress-icon.png with your own icons
  • Integration Logos: Update SVG files in public/logos/ if needed

4. Modifying Sections

Adding a New Section

  1. Copy an existing section structure from index.html
  2. Paste it where you want the new section
  3. Update the content and IDs
  4. Add corresponding styles in styles.css (or reuse existing classes)

Removing a Section

  1. Find the section in index.html (look for HTML comments)
  2. Delete the entire <section> block
  3. Remove any navigation links pointing to that section

5. Updating Links

Update all links in index.html:

  • Navigation links: Update href attributes in the navbar
  • CTA buttons: Change "Start Free Trial" button links
  • Social links: Update footer social media URLs
  • Footer links: Update all footer navigation links

üÜò Support & Help

Getting Help

If you encounter any issues or have questions about customizing the template:

  • Check this documentation first - most common questions are answered here
  • Review the code comments in index.html and styles.css
  • Inspect the browser's developer tools (F12) to see how styles are applied

Common Issues & Solutions

Images Not Displaying

  • Check that image file paths are correct (case-sensitive on some servers)
  • Ensure images are in the correct folders
  • Verify image file names match exactly (including extensions)

Styles Not Applying

  • Clear browser cache (Ctrl+F5 or Cmd+Shift+R)
  • Check that styles.css is linked correctly in index.html
  • Verify CSS syntax is correct (no unclosed brackets)

JavaScript Not Working

  • Open browser console (F12) to check for JavaScript errors
  • Ensure script.js is linked before closing </body> tag
  • Check that jQuery or other dependencies aren't required (this template uses vanilla JS)

Responsive Issues

  • Test on actual devices, not just browser resize
  • Check viewport meta tag is present: <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Verify media queries in styles.css are correct

Browser Compatibility

This template is tested and works in:

  • Chrome (latest)
  • Firefox (latest)
  • Safari (latest)
  • Edge (latest)
  • Mobile browsers (iOS Safari, Chrome Mobile)
Note: Some older browsers (Internet Explorer) are not supported. The template uses modern CSS features that require up-to-date browsers.
